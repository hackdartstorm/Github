## ğŸ“‹ Pull Request Guidelines

### Before Submitting a Pull Request

1. **Check for existing PRs** - Make sure there isn't already a PR that addresses the same issue
2. **Test your changes** - Ensure all links work and formatting is correct
3. **Follow the style guide** - Match the existing tone and format
4. **Update documentation** - If you add new content, update the Table of Contents

### Pull Request Process

1. Fork the repository
2. Create a new branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Commit with clear messages:
   ```bash
   git commit -m "Add: new section on git rebase"
   ```
5. Push to your branch:
   ```bash
   git push origin feature/your-feature-name
   ```
6. Open a Pull Request
